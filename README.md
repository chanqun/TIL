TDD사이클은
1. 실패하는 테스트 코드를 만든다.
2. 테스트를 성공시킨다.
3. 리팩토링한다.

> 리팩토링이란?  
> 코드의 행위(behavior)를 수정하지 않은 채로 내부 구조(structure)를 다듬는 것
> - 리팩토링과 테스트는 땔 수 없는 관계이다.
